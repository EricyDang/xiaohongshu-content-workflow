# Brand And Visual System

Use this reference when the user provides an account persona, brand, product, visual identity, or authorized original character. Do not invent or preserve private IP details unless the user explicitly asks for them.

## Required Inputs
- Account positioning
- Target audience
- Content goal
- Tone words
- Visual subject
- Allowed assets
- Forbidden elements
- Rights or usage limits

If any input is missing, infer a neutral direction and label it as an assumption.

## Visual Consistency Rules
- Keep the same subject, palette, line weight, layout logic, and typography direction across a post series.
- Use enough blank space for mobile reading.
- Keep cover hierarchy obvious: title first, visual subject second, supporting copy third.
- Do not add logos, watermarks, trademarks, or character likenesses unless the user owns or provides them.
- Prefer original visual systems built from mood, audience, product, scene, and palette.

## Common Style Directions
- Clean lifestyle photo
- Minimal editorial layout
- Hand-drawn healing illustration
- Dense study-note layout
- Product detail close-up
- Step-by-step tutorial board
- Checklist or worksheet style
- Light comic strip

## Rights Check
Before final delivery, check:

- The package does not contain private personal IP.
- Any provided brand or character is clearly user-authorized.
- No celebrity, logo, trademark, or copyrighted character is copied without permission.
- No private user information appears in screenshots, resumes, chat records, or examples.
